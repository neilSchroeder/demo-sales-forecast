# Contributing to Sales Forecasting Portfolio Project

Thank you for your interest in contributing to this project!

## Development Setup

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/yourusername/demo-sales-forecast.git
   cd demo-sales-forecast
   ```
3. Create a virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```
4. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

## Code Style

- Follow PEP 8 guidelines for Python code
- Use meaningful variable and function names
- Add docstrings to all functions and classes
- Keep functions focused and modular

## Testing

- Test any changes to data processing or forecasting modules
- Verify that notebooks run without errors
- Check that visualizations render correctly

## Submitting Changes

1. Create a new branch for your feature:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your changes and commit:
   ```bash
   git add .
   git commit -m "Add your descriptive commit message"
   ```
3. Push to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
4. Open a Pull Request with a clear description of your changes

## Areas for Contribution

- Additional forecasting models (ARIMA, SARIMA, LSTM)
- Category-specific or regional forecasting
- Dashboard improvements
- Documentation enhancements
- Performance optimizations
- Additional data sources

Thank you for contributing!
